# AGENTS.md

LEET 데일리 — 로스쿨 준비생을 위한 "하루 한 문제" LEET 풀이 사이트.

이 파일은 코딩 에이전트(Codex 등)가 이 저장소에서 작업할 때 따라야 할 규칙을 정의한다. 제품 요구사항, 화면 구성, 데이터 모델, API 설계, 로드맵은 `PROJECT.md`를 기준으로 한다.

기준 기획 문서: Notion "[WEEK16] 기획". `PROJECT.md`와 기획 문서가 충돌하면 기획 문서가 우선한다.

## 1. 프로젝트 기준 문서

- 제품/기술 설계의 상세 기준은 `PROJECT.md`를 따른다.
- 작업 전 관련 기능의 비즈니스 규칙, 데이터 모델, API 계약을 `PROJECT.md`에서 확인한다.
- 이 파일은 에이전트 작업 방식, 코딩 컨벤션, 구현 시 금지사항을 우선 정의한다.

## 2. 기술 스택 요약

- 프론트엔드: React, TailwindCSS
- 백엔드: FastAPI (Python)
- ORM/마이그레이션: SQLAlchemy, Alembic
- DB: PostgreSQL
- 인증: 소셜 OAuth (구글) + 자체 JWT

## 3. 디렉터리 구조 기준

```
/backend
  /app
    /models        # SQLAlchemy 모델
    /schemas       # Pydantic 스키마
    /api           # 라우터 (auth, daily, attempts, board, stats, settings)
    /core          # 설정, 보안(JWT), 의존성
    /services      # 비즈니스 로직 (스트릭, 문제 배정, 오답 복습 스케줄링)
    main.py
  /alembic         # 마이그레이션
/frontend
  /src
    /pages         # Login, Main, Board, MyPage
    /components
    /api           # 백엔드 호출 클라이언트
    /hooks
```

## 4. 코딩 컨벤션

- 스키마 변경은 항상 Alembic 마이그레이션으로 처리한다. 모델 직접 수정 후 마이그레이션 누락 금지.
- DB 접근/비즈니스 로직은 `/services` 또는 라우터 의존성 주입을 통한다. 라우터에 로직을 직접 두지 않는다.
- 정답(`answer_index`)·해설(`explanation`)은 채점 전 응답에 절대 포함하지 않는다.
- 게시판 응답은 스포일러 차단 규칙(미제출 403)을 서버에서 강제한다. 프론트 숨김에만 의존하지 않는다.
- 프론트는 TailwindCSS 유틸리티 클래스 사용. 인라인 스타일 지양.
- 비밀키/토큰은 환경변수로 관리하고 커밋하지 않는다.

## 5. 구현 시 반드시 지킬 비즈니스 제약

- 오늘의 문제는 사용자별, 날짜별로 고정한다. 같은 날 새로고침해도 바뀌면 안 된다.
- 문제 배정은 사용자별 개인 진도 기준이며 전역 공통 배정으로 구현하지 않는다.
- 스트릭은 데일리 문제 완료만 반영한다. 추가 연습은 스트릭에 반영하지 않는다.
- 제출/채점 전 `reasoning`이 비어 있으면 서버에서 거부한다.
- 제출 시 reasoning은 해당 문제 게시판에 자동 등록한다. 한 사용자는 한 문제당 게시물 1개만 갖는다.
- 문제별 게시판 조회는 서버에서 제출 여부를 확인해 미제출 사용자에게 403을 반환한다.
- 신규 사용자 가입 시 `user_settings` 기본 행을 함께 생성한다.
- 데일리 오답 복습 스케줄링은 `user_settings.review_interval_days`를 따른다. 값이 `null`이면 스케줄링하지 않는다.

## 6. 중요 제약 / 주의사항

- LEET 기출 자료 사이트(`leet.uwayapply.com`)에 대한 자동 스크래핑을 구현하지 않는다.
- 데이터는 수동 다운로드 → 파싱 → 검수 파이프라인으로 적재한다.
- LEET 기출 저작권 및 이용 허락 범위를 고려한다. 외부 공개 서비스화 전에는 권리 범위를 확인해야 한다.
- 비밀키, OAuth 토큰, JWT 시크릿, DB 비밀번호는 커밋하지 않는다.
